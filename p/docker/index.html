Docker简介

Docker是一个开源的容器化平台，它允许开发人员和运维人员在轻量级的虚拟化环境中构建、部署和运行应用程序。可以将Docker想要成一个货运集装箱系统：货物（应用程序）被打包进标准化的集装箱（容器）中，不论是用卡车、火车还是轮船（不同操作系统或云环境），都能轻松运输和运行。

Docker的核心概念

1.镜像(Image)

  • 镜像是一个只读的模板，类似于快递公司打包货物时用的“包装规范”。它包含了运行应用所需的所有东西（代码、库、依赖）。

比喻：镜像就像一个食谱，你根据这个食谱（镜像）可以多次制作同样的菜（容器）。

2.容器（Container)

  • 容器是镜像的运行实例，就像按照食谱做出的具体的一道菜。

  • 容器是独立的，彼此互不影响。

比喻：容器就像一个租来的公寓，每个公寓都装修好了（镜像），可以住人（运行应用）。

3.Docker引擎（DockerEngine）

  • Docker引擎是运行容器的底层引擎，相当于“集装箱运输的物流系统”。

4.Dockerfile

  • Dockerfile是一个定义镜像构建过程的文本文件。

比喻：Dockerfile 就像是“货物打包清单”，告诉工人（Docker 引擎）如何一步步打包货物（构建镜像）。

5.DockerHub

DockerHub 是一个镜像仓库，就像是快递公司的仓库，存储了各种打包好的标准集装箱（镜像），随时可以调取。

Docker的优点

  1. 轻量化
  • 与虚拟机不同，Docker容器共享宿主机的操作系统内核，就像把一个大货物拆成许多个小包裹运输，节省了资源。
  1. 可移植性
  • Docker的标准化使得在开发环境、测试环境和生产环境运行的代码完全一致，就像无论你在哪个国家的码头，集装箱都能被吊装和运输。
  1. 快速部署
  • 启动容器速度非常快，就像打开一个已经打包好的行李箱即可使用。
  1. 版本控制
  • Docker支持为镜像打标签，像软件版本一样管理。
  1. 隔离性
  • 每个容器之间互不影响，就像每个租来的公寓门一关，里面的事情彼此无关。

Docker的基本操作

1.镜像操作

  • 拉取镜像：docker pull<镜像名>（从仓库取一个装满货物的集装箱）

  • 列出镜像：docker images（查看仓库里有哪些可用的集装箱）

  • 删除镜像：docker run<镜像ID>（删除某个不再需要的集装箱模板）

2.容器操作

  • 启动容器：docker run - d - p 8080:80<镜像名>（将集装箱吊装到卡车上，启动运输）

  • 列出运行容器：docker ps

  • 停止容器：docker stop <容器ID>（将集装箱从卡车上卸下，停止运行）

  • 删除容器：docker rm <容器ID>（销毁一个不再需要的集装箱实例）

3.构建镜像

  • 使用Dockerfile构建镜像：docker build - t <镜像名>（根据食谱制作一个全新的菜品）

4.调试容器

  • 查看容器日志：docker logs <容器ID>（检查运输过程中发生了什么）

  • 进入容器：docker exec - it <容器ID> /bin/bash（进入集装箱内部，检查货物情况）

Docker的应用场景

1.微服务架构

  • 每个微服务运行在独立的容器中，就像每个快递包裹都独立包装、单独配送，互不干扰。

2.CI/CD流程

  • Docker容器在持续集成和持续部署流程中使用，就像标准化的货物清单，确保每次交付都是一致的。

3.跨平台部署

  • 使用Docker，可以将应用程序在不同平台（Windows、Linux、云服务器）上无缝运行。

比喻：无论你用的是什么交通工具（环境），标准的集装箱都能被运输。

4.开发与测试

  • Docker提供了隔离的开发环境，避免不同开发者的环境差异导致问题。比喻：就像每个开发者都用自己的专属工作室（容器），外界变化不会影响开发过程。

示例：用Docker构建一个简单的应用

  1. 创建一个 Dockerfile：
1
2
3
4
5

FROM python:3.9-slim
WORKDIR /app
COPY . /app
RUN pip install -r requirements.txt
CMD ["python", "app.py"]

比喻：这份 Dockerfile 就像一个“打包说明”，告诉 Docker 如何准备应用程序。

  1. 构建镜像：
1

docker build -t my-python-app .

比喻：这一步是“根据食谱（Dockerfile）做好菜（镜像）”。

  1. 运行容器：
1

docker run -d -p 5000:5000 my-python-app

比喻：运行容器相当于“端上桌，让别人来吃这道菜”。

  1. 查看结果：
  • 打开浏览器访问 http://localhost:5000

比喻：就像客人根据菜单，享用你端上来的美食。

安装与部署(windows)

系统要求

首先就需要安装 Hyper-V或 WSL2这个windows自带的功能（建议WSl2）

  1. 任务栏搜索功能，启用"适用于Linux的Windows子系统" + “虚拟机平台”

  2. 管理员权限打开命令提示符，安装wsl2

1
2

wsl --set-default-version 2
wsl --update --web-download

  1. 安装dockerDesktop

  2. 配置镜像，建议开梯子拉镜像 QAQ

重要事项

要运行Windows容器，您需要Windows10或Windows11专业版或企业版。Windows家庭版或教育版仅允许您运行Linux容器。
